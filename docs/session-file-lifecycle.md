# Session File Lifecycle

How and when JSONL session files are created, rotated, and archived on disk.

## File Layout

```
{sessions.dir}/
  {agentID}/{type}{id}/{versionTS}/root.jsonl          ← root session
  {agentID}/{type}{id}/{versionTS}/b{childTS}.jsonl    ← branch
  {agentID}/{type}{id}/{versionTS}/i{childTS}.jsonl    ← independent spawn
```

- `type` = `c` (chat) or `i` (independent)
- `id` = chat ID (e.g. `123456789`) or timestamp (for independent sessions)
- `versionTS` = Unix timestamp — set at creation, updated on compaction rotation

Key → path mapping is in `store.go:SessionPath`:
- Root keys (last segment is pure number) → `{dir}/{key}/root.jsonl`
- Child keys (last segment starts with `b` or `i`) → `{dir}/{key}.jsonl`

## When New Files Are Created

### 1. Root session: first append (lazy creation)

**Trigger:** First call to `Append()` or `AppendAll()` on a session key that has no file on disk.

**Code path:** `store.go:appendUnlocked` (line 252)

```
os.Stat(path) → not found
os.OpenFile(path, O_APPEND|O_CREATE|O_WRONLY)
write session_meta line → {"type":"session_meta","created_at":"..."}
write message
```

A new session key is generated by `NewChatSessionKey(agentID, chatID)` which calls `NewChatSession` → `SessionKey{..., VersionTS: time.Now().Unix()}`. The key is cached in-memory (`Bot.chatSessionKeys`) and persisted to the session index via `SetChatMetadata`.

**Important:** `RotateKey` and `ReplaceAndRotate` generate new session keys, but `RotateKey` does NOT create a file — it only archives the old one. The new file is created lazily on the next `Append`.

### 2. Branch file: immediately on branch creation

**Trigger:** `CreateBranchWithOptions(parentKey, branchKey, opts)` in `branch.go`.

**Code path:** `branch.go:CreateBranchWithOptions` (line 29)

```
os.Create(path)
write branch_meta line → {"type":"branch_meta","parent_key":"...","branch_point":N}
optionally write orientation message
```

Branch keys are derived from the parent: `parentKey.Branch()` → appends `/b{time.Now().Unix()}` to the parent key. The file is created eagerly (not lazily).

### 3. Compaction: ReplaceAndRotate creates new file + archives old

**Trigger:** Compactor calls `writer.ReplaceAndRotate(oldKey, compactedMessages)` in `compact.go:544`.

**Code path:** `archive.go:ReplaceAndRotate` (line 158)

```
1. Read branch_meta/session_meta from old file
2. rotateKeyInternal(oldKey):
   - newKey = oldKey.WithVersion(time.Now().Unix())   ← NEW versionTS
   - os.Rename(oldPath, nextArchivePath(oldPath))     ← archive old file
3. os.Create(newPath)                                  ← new path with new versionTS
4. Write preserved session_meta (original creation time)
5. Write compacted messages
```

**Result:**
- Old file: renamed to `root.2026-03-13T10-30-00Z.jsonl` (in same directory)
- New file: `{agentID}/{type}{id}/{NEW_versionTS}/root.jsonl` (new directory)
- Session key changes: e.g. `bot/c123/1710000000` → `bot/c123/1710340200`

The new key is propagated to the bot via `SessionKeyRotatedFunc` → `UpdateChatSessionKey(chatID, newKey)`, which updates both the in-memory cache and the persisted session index.

### 4. Reset (/reset command): RotateKey + lazy creation

**Trigger:** `/reset` command calls `RotateKey(oldKey)`.

**Code path:** `archive.go:RotateKey` (line 234)

```
1. rotateKeyInternal(oldKey):
   - newKey = oldKey.WithVersion(time.Now().Unix())
   - os.Rename(oldPath, nextArchivePath(oldPath))
2. Returns newKey — NO file created yet
3. Next Append() creates the file lazily
```

### 5. Replace (in-place compaction for branches)

**Trigger:** `replaceInternal(key, msgs)` in `archive.go:253`.

```
1. os.Rename(path, nextArchivePath(path))   ← archive old
2. os.Create(path)                           ← same path, same key
3. Write metadata + compacted messages
```

Key does NOT change. Old file is archived with timestamp suffix.

## On Service Restart

**The existing session continues appending to the same file. No new root file is created.**

On startup (`sessions_init.go:initSessions`):

1. `RepairOrphans()` — walks all `.jsonl` files, finds assistant messages ending with tool_use blocks that have no tool_result response. Appends synthetic "Tool call interrupted" tool_result messages to the **existing file**.

2. `InjectRestartMarkers()` — walks all `.jsonl` files modified within `RestartMarkerMaxAge` (1 hour). Appends a "SYSTEM RESTART" user message to the **existing file**.

3. `SessionIndex.Rebuild(store)` — scans all session files on disk and rebuilds the SQLite index.

4. `chatmeta.Resolver.SessionKeyForChat(chatID)` — on first message after restart, queries `SessionIndex.GetChatMetadata(agentID, chatID, "session_key")` which returns the **persisted key** from before restart. Continues using it.

**Net effect:** The same `root.jsonl` file keeps getting appended to across restarts. No new file, no new key. The restart is only visible as injected marker messages within the file.

## Why Your Session Is All In One File

Given the behavior above, a session that has been running for hours with restarts will have **everything in a single `root.jsonl`** because:

1. Restart does NOT rotate/create a new file — it appends restart markers to the existing one
2. The session key is persisted in the SQLite index and restored on startup
3. Only compaction creates a new root file (with a new `versionTS` directory)
4. Only `/reset` creates a new root file (lazily, on next message)

If compaction hasn't triggered, and no one ran `/reset`, that single `root.jsonl` contains every message since the session was first created.

### If grep can't find recent content

The file is append-only JSONL — recent messages are at the **end** of the file. Check:
- `tail -n 50 /path/to/root.jsonl | jq .` to see recent messages
- The file may be very large; grep may be matching earlier content
- Messages are JSON-encoded, so content may have escaped characters

## Archive Lifecycle

```
root.jsonl (active)
  ↓ compaction
root.2026-03-13T10-30-00Z.jsonl (archived, renamed)
  + new versionTS directory/root.jsonl (new active)
  ↓ ArchiveSweep (every 6h, for sessions idle > archive_after)
root.2026-03-13T10-30-00Z.jsonl.gz (gzipped, original deleted)
```

- `ArchiveSweep` runs every 6 hours + on startup
- Only archives sessions whose last activity is older than `sessions.archive_after`
- Never archives the current session for each agent+chat
- Never archives sessions with active branches
- Gzipped files are transparently decompressed on `Load()` if needed

## Summary Table

| Event | New File? | New Key? | Old File |
|---|---|---|---|
| First message in new chat | Yes (lazy) | Yes (new versionTS) | n/a |
| Normal message | No (append) | No | n/a |
| Service restart | No (append markers) | No (restored from index) | n/a |
| Compaction (ReplaceAndRotate) | Yes (new versionTS dir) | Yes | Renamed to `.{timestamp}.jsonl` |
| /reset (RotateKey) | Yes (lazy, on next msg) | Yes | Renamed to `.{timestamp}.jsonl` |
| Branch creation | Yes (immediate) | Yes (child key) | n/a |
| Replace (branch compaction) | Yes (same path) | No | Renamed to `.{timestamp}.jsonl` |
| ArchiveSweep | No | No | Gzipped to `.jsonl.gz`, original deleted |
